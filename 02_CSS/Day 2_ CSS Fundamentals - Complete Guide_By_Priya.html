1. Introduction to CSS

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of HTML documents. It controls how elements look on the page.

Key Concepts:

  • Cascading: Multiple styles can apply, with rules determining priority
  • Style: Visual appearance (colors, fonts, layout)
  • Sheets: Can be external files or embedded in HTML

What CSS Can Do:

Layout & Positioning

Control element placement and flow

Colors & Backgrounds

Set colors, gradients, and images

Typography & Text

Style fonts, sizes, and spacing

Animations & Transitions

Create movement and effects

3. Ways to Add CSS

Method 1: Inline CSS (Not Recommended)

<h1 style="color: red; font-size: 24px;">Heading</h1>

Method 2: Internal/Embedded CSS

<head>
    <style>
        h1 {
            color: blue;
            font-family: Arial;
        }
    </style>
</head>

Method 3: External CSS (Recommended)

<head>
    <link rel="stylesheet" href="styles.css">
</head>

CSS Priority (Specificity):

  1. Inline styles (style attribute) - Highest priority
  2. ID selectors (#id)
  3. Class selectors (.class)
  4. Element selectors (p, div, h1)
  5. Browser default styles - Lowest priority

4. CSS Selectors

8. Display Property

Display Values:

/* Block - Full width, new line */
.block { display: block; }

/* Inline - Flows with text */

9. Positioning

Position Values:

/* Static - Default, normal flow */

/* Sticky - Hybrid of relative and fixed */

12. Responsive Design

Viewport Meta Tag:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries:

/* Mobile First Approach */
.container {
    padding: 1rem;
}

/* Tablet */
@media (min-width: 768px) {
    .container {
        padding: 2rem;
    }
}

/* Desktop */
@media (min-width: 1024px) {
    .container {
        max-width: 1200px;
        margin: 0 auto;
    }

13. CSS Variables (Custom Properties)

Declaring Variables:

:root {
    /* Global variables */
    --primary-color: #3498db;
    --spacing-unit: 1rem;

Using Variables:

    background-color: var(--primary-color);
    padding: calc(var(--spacing-unit) * 1.5);
    border-radius: var(--border-radius);

14. Transitions and Animations

CSS Transitions:

.button {
    transition: all 0.3s ease;
    /* property | duration | timing-function | delay */
}

.button:hover {
    transform: scale(1.1);
    background-color: #2ecc71;
}

CSS Animations:

@keyframes slideIn {
    from { transform: translateX(-100%); }
    to { transform: translateX(0); }
}

.element {
    animation: slideIn 1s ease;
    /* name | duration | timing-function | delay | count */
}

15. CSS Best Practices

1. Organization & Structure:

/* Recommended order within ruleset */
.element {
    /* Positioning */
    position: relative;
    
    /* Display & Box Model */
    display: block;
    width: 100%;
    padding: 1rem;
    
    /* Typography */
    font-family: sans-serif;
    font-size: 16px;
    
    /* Colors & Background */
    color: #333;
    background-color: #fff;
}

2. Naming Conventions (BEM):

/* Block */
.card {}

/* Element */
.card__title {}

/* Modifier */
.card--featured {}

3. Performance Tips:

  • Use transform and opacity for animations (they don't trigger layout recalculation)
  • Minimize the use of expensive properties like box-shadow and border-radius on many elements
  • Use will-change to hint browsers about upcoming animations

4. Accessibility:

/* Focus styles for keyboard navigation */
button:focus,
a:focus,
input:focus {
    outline: 3px solid #4d90fe;
    outline-offset: 2px;
}
